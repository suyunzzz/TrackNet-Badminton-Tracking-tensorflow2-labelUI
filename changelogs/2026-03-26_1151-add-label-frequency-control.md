# 变更记录

## 概要
- 为标注流程新增 `--label_hz` 参数，默认按 `10Hz` 控制标注频率，即相邻标注帧默认间隔约 `0.1s`。
- 调整网页标注与桌面标注的导航逻辑，使“上一帧 / 下一帧”按采样后的标注步长移动，而不是逐帧移动。
- 更新文档与前端文案，补充标注频率、步长显示和新的快捷键说明。

## 修改文件
- `parser.py`：新增 `--label_hz` 命令行参数，供标注工具统一使用。
- `web_label.py`：根据视频 `fps` 和 `label_hz` 计算标注步长，按采样帧统计进度，并让“下一未标注”按标注步长跳转。
- `web_label_static/app.js`：网页端按标注步长进行前进、后退和快速跳转，并显示当前频率与步长。
- `web_label_static/index.html`：更新页面文案，展示标注频率信息，并将导航按钮说明改为“标注帧”。
- `imgLabel.py`：桌面标注工具改为按标注步长移动，并输出当前频率与步长信息。
- `README.md`：补充 `--label_hz` 的说明，并更新网页端与桌面端的使用描述。
- `requirements.txt`：将当前网页标注不必需的 `focal-loss` 和 `tensorflow` 注释掉，保留最小依赖说明。

## 行为变化
- 对于 `29.97fps` 的视频，默认 `10Hz` 会换算为每 `3` 帧标注一次。
- 网页端“下一未标注”和进度统计只针对采样后的目标标注帧，不再按全部原始帧统计。
- 桌面端 `n / p / > / <` 的移动步长会随 `label_hz` 自动变化。
