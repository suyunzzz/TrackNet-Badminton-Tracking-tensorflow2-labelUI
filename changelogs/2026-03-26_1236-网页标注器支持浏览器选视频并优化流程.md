# 变更记录

## 建议的 Commit Message

`Add browser video picker for web labeler`

## 本次变更概述

- 将网页标注工具从“启动时必须传入视频路径”改为“启动后可在浏览器里直接选择视频”。
- 为网页标注页增加空状态、欢迎引导和更清晰的主操作按钮，降低首次使用门槛。
- 标注后自动跳到下一采样帧，配合现有按采样频率标注的逻辑，减少重复点击。
- 新增一个便捷的 Windows 启动脚本，并补充一份新的测试标注数据。

## 详细变更

### 1. 后端支持运行时选择视频

- `web_label.py` 新增 `choose_video_file()`，通过 `tkinter` 文件选择框让用户直接挑选 `.mp4`。
- 新增 `SessionManager`，把“未加载视频”和“已加载会话”两种状态统一收口，避免请求处理器直接依赖固定 session。
- 新增 `/api/select_video` 接口，并让 `/api/state` 在未选择视频时返回 `ready: false` 的空状态。
- 启动服务后会自动打开浏览器；如果命令行已传入合法视频，仍可沿用原来的预加载模式。

### 2. 前端改成先选视频再进入工作区

- `web_label_static/index.html` 重构为“欢迎区 + 空状态 + 工作区”三段式布局，主流程改成先点“选择视频开始”。
- `web_label_static/app.js` 增加 `ready` 状态切换、选择/更换视频按钮绑定，以及未加载视频时的交互保护。
- 前端在加载视频成功后自动切入标注界面并显示首帧，不再要求手动输入视频路径。
- 保存提示、错误提示和状态文案进一步中文化。

### 3. 标注操作与视觉体验优化

- `web_label_static/app.js` 在“标有球”和“标记无球”后都会自动跳到下一采样帧，提升连续标注效率。
- 导航按钮和快捷键继续按采样步长工作，快进快退保持以 `36 * step` 为单位。
- `web_label_static/styles.css` 整体更新了配色、布局、空状态、侧栏面板和移动端适配，让页面更像面向普通用户的成品工具。

### 4. 附带文件变化

- `打开标注工具.vbs` 新增了一个静默启动脚本，会调用上级目录 `venv` 中的 Python 来运行 `web_label.py`。
- `test/test.csv` 有少量标注坐标与是否有球的结果调整。
- `test/test2.csv` 新增了一份测试标注数据，内容上看与本次新增测试视频属于同一组素材。
